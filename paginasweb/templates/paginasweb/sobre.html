Descrição do projeto

Objetivo geral e funcionalidades principais

Este projeto tem como objetivo o desenvolvimento de um sistema web para uma loja virtual de multi-nichos, que comercializa uma variedade de produtos em categorias distintas como utilidades domésticas, itens pet, maquiagem e decoração. A estrutura do sistema visa oferecer uma navegação intuitiva por meio da organização hierárquica de categorias e subcategorias, facilitando a busca e o acesso aos produtos.

Objetivo Geral

Criar uma plataforma e-commerce funcional e escalável, que permita ao administrador cadastrar produtos e gerenciar pedidos, e ao cliente visualizar e comprar produtos de forma simples, rápida e segura.

Objetivos Específicos

  • Criar uma estrutura hierárquica de categorias e subcategorias para organização dos produtos;
  • Permitir o cadastro, edição e exclusão de produtos vinculados a subcategorias específicas;
  • Gerenciar estoque e preços dos produtos cadastrados;
  • Permitir que usuários se cadastrem, realizem login e efetuem compras no sistema;
  • Registrar e controlar os pedidos realizados pelos usuários, com acompanhamento de status (pendente, pago, enviado, entregue, etc.);
  • Oferecer uma interface clara e acessível tanto para clientes quanto para administradores.

Funcionalidades principais

  • Cadastro e gerenciamento de produtos e categorias;
  • Cadastro de usuários com distinção entre clientes e administradores;
  • Criação e controle de pedidos com registro de itens, quantidade e valor total;
  • Acompanhamento do status dos pedidos;
  • Validação de dados como CPF e e-mail únicos;
  • Modelagem clara com uso de relações entre classes (Categoria, SubCategoria, Produto, Pedido, etc.).

Diagrama de Caso de Uso

Um Diagrama de Caso de Uso é uma ferramenta visual que descreve a funcionalidade de um sistema do ponto de vista do usuário (ou "ator"). Ele mostra:

  • Atores: Quem ou o quê interage com o sistema.
  • Casos de Uso: As tarefas específicas que os atores podem realizar com o sistema.
  • Relacionamentos: Como os casos de uso se relacionam entre si e com os atores (ex: inclusão, extensão, generalização).

Em resumo, ele responde à pergunta: "O que o sistema faz?" e para quem. É útil para entender os requisitos do sistema e definir seu escopo.

Diagrama de Classes

Um Diagrama de Classes é um tipo de diagrama estrutural da UML (Linguagem Unificada de Modelagem) que descreve a estrutura estática de um sistema, mostrando suas classes, atributos, métodos e relacionamentos entre elas. Ele mostra:

  • Classes: Representam os objetos do sistema e seus dados.
  • Atributos: As características dos objetos (ex: nome, idade).
  • Métodos: As ações que os objetos podem realizar (ex: calcular, exibir).
  • Relacionamentos: Como as classes se relacionam entre si (ex: associação, herança, agregação, composição).

Em resumo, ele responde à pergunta: "Como o sistema é estruturado?". É útil para modelar a arquitetura do software e facilitar a comunicação entre desenvolvedores.
